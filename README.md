# Прогнозирование ресурсных потребностей стационара методами машинного обучения

Прогнозирование ресурсной нагрузки от пациента **в момент поступления** в стационар — на основе административных данных медицинской информационной системы (МИС), без использования лабораторных показателей.

Проект выполнен в рамках выпускной квалификационной работы (КФУ, Институт физики, направление 12.03.04 «Биотехнические системы и технологии», 2026).

---

## Задача

Российские стационары финансируются по системе ОМС за законченный случай лечения, а не за проведённые дни. Затяжная госпитализация снижает оборот коечного фонда и доход учреждения. Однако в момент поступления пациента администрация не знает, насколько ресурсоёмким окажется его лечение.

Цель проекта — построить модель, которая по данным, доступным **при поступлении**, прогнозирует ресурсную нагрузку. Задача разбита на две независимые задачи бинарной классификации:

- **Задача A — затяжная госпитализация (PLoS):** длительность пребывания > 7 суток (порог по Zeleke et al., 2023)
- **Задача Б — сложное лечение:** потребность в операции или госпитализации в ОРИТ

---

## Данные

- **5 868** случаев госпитализации детского стационара
- **849** уникальных кодов диагнозов по МКБ-10
- **41** профиль отделения
- **14** признаков, доступных при поступлении (демографические, клинические, диагностические, временные)

> Исходные данные защищены соглашением о конфиденциальности (NDA) и в репозиторий не включены. Структура данных и логика обработки описаны в коде.

Принципиальное ограничение постановки: используются **только административные данные**, доступные при первичной регистрации. Лабораторные показатели намеренно исключены — это делает модель применимой в любом стационаре без интеграции с лабораторными системами.

---

## Методы

Сравнивались четыре алгоритма классификации:

| Модель | AUC-ROC (Задача A) | AUC-ROC (Задача Б) |
|--------|:------------------:|:------------------:|
| Logistic Regression | 0,701 | 0,784 |
| Random Forest | 0,808 | 0,948 |
| Gradient Boosting | 0,800 | 0,957 |
| **CatBoost** ⭐ | **0,826** | **0,963** |

**CatBoost** показал наилучший результат благодаря нативной обработке категориальных признаков (упорядоченное целевое кодирование), что критично при 849 уникальных кодах МКБ-10.

Применённые техники:
- стратифицированное разбиение выборки (80/20)
- 5-кратная стратифицированная кросс-валидация
- подбор гиперпараметров случайным поиском (random search, 30 итераций)
- ранняя остановка (early stopping) для защиты от переобучения
- взвешивание классов для компенсации дисбаланса

### Метрики лучшей модели (CatBoost)

| Метрика | Задача A (PLoS) | Задача Б (сложное) |
|---------|:---------------:|:------------------:|
| Accuracy | 0,744 | 0,914 |
| Precision | 0,76 | 0,82 |
| Recall | 0,76 | 0,91 |
| F1-score | 0,76 | 0,86 |
| AUC-ROC | 0,826 | 0,963 |
| CV AUC (5-fold) | 0,830 ± 0,004 | 0,958 ± 0,008 |

### Важность признаков

Наиболее информативные признаки (Задача A):
- **Профиль отделения — 28,1%**
- Диагноз по МКБ-10 (3 уровня детализации) — суммарно 37,3%
- Тяжесть состояния — 6,8%
- Возраст — 5,5%

Профиль отделения + диагноз дают более 65% предсказательной силы.

## Веб-приложение

На основе обученных моделей разработан прототип веб-приложения на **Streamlit**. Оператор вводит данные пациента при поступлении, приложение в реальном времени возвращает два прогноза с рекомендациями по резервированию ресурсов.

### Запуск

```bash
pip install -r requirements.txt
streamlit run app.py
```

Приложение откроется на `http://localhost:8501`.

> Для работы приложения нужны файлы обученных моделей (`model_plos.cbm`, `model_complex.cbm`, `model_meta.json`), которые создаются после запуска `diploma_binary.py`.
